# Issue Tracker

This repository uses GitHub Issues as the source of truth for work tracking.

Use the GitHub CLI (`gh`) or the GitHub app workflow available in Codex to inspect, create, update, and triage issues.

Guidelines:

- Treat an issue as the primary record of a task, bug, or feature request.
- Link pull requests back to the issue they resolve when practical.
- Prefer updating the existing issue rather than creating duplicates.
- If a task is too large, split it into smaller issues that can be worked independently.
- Use the repository's configured labels and triage flow rather than inventing new label names ad hoc.
